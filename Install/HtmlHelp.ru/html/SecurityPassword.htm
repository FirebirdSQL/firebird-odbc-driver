Защита Пароля

При записи DSN через ODBC Администратор пароль шифруется и записывается в ODBC.INI. Однако пароль допускается не указывать при формировании DSN. Он будет запрошен дополнительно в момент соединения с базой данных или может быть передан в строке соединения. Для получения динамически шифрованный вариант пароля, необходимо в операторе соединения SQLDriverConnect указать буфер для выходного параметра SQLCHAR *OutConnectionString, длина которого должна быть не меньше 512, желательно 1024.
