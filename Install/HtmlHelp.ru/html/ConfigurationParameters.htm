Параметры конфигурации Firebird ODBC

Используются, чтобы определить параметры соединения для базы данных. Диалоговое окно содержит параметры, которые соответствуют атрибутам соединения

Имя Источника Данных(DSN)

Обязательный параметр.
Уникальное имя среди остальных созданных DSN, желательно упомянуть имя сервера для упрощения выбора соединения.
Например: “Соединение с FbEmbed” или “ConnectFbServer”

Примечание

Не обязательный параметр.
Более детализированное описание источника данных.

Драйвер

Обязательный параметр.
Всегда: IscDbc

База данных

Обязательный параметр.
Определите расположение базы данных, локально, дистанционно или через псевдоним.
Смотри примеры

Клиент

Обязательный параметр, когда используется внедренный сервер. Позволяет Вам явно указать расположение Firebird внедренный сервер (fbembed) или Firebird SQL клиента (gds32,fbclient). Однако, необходимо подчеркнуть, для обычного сервера этот параметр можно не указывать.

Идентификатор пользователя или Логин

Не обязательный параметр.
Имя пользователя, которое нужно использовать при соединении с Firebird базой данных. Если не указано, ODBC запросит Вас относительно (UID или USER) пользователя непосредственно при соединении с источником данных.

Пароль

Не обязательный параметр.
Пароль, который нужно использовать с идентификатором пользователя (UID или USER) при соединении с Firebird базой данных. Если не указано, ODBC запросит Вас относительно пароля(PWD or PASSWORD ) при соединении с источником данных. Если используется, пароль автоматически будет зашифрован и сохранен в odbc.ini. Так что при указании пароля нет риска защиты.

Роль

Не обязательный параметр.
Строка длиной до 31 символа, выражающая роль пользователя при соединении с базой данных. Пользователь добавляет себе привилегии указанной роли.

Символьный набор

Не обязательный параметр.
Установите заданный по умолчанию символьный набор.

Опции

    Транзакция
    Определите тип транзакции при соединении с Firebird базой данных

      Читать (по умолчанию писать)
      Писать: Доступ к базе в режиме чтение/запись.
      Читать: Доступ к базе в режиме только чтение.

      Не ждать (по умолчанию: ждать)
      Не ждать: транзакция немедленно возвратит ошибку, в случае когда ее доступ к данным блокирован незавершенной транзакцией.
      Ждать: транзакция не начнет свою работу, пока доступ к данным будет блокированн другой незавершенной транзакцией.

        Время блокировки(для случая: ждать)
        Время в секундах, по истечению которого будет возвращена ошибка: lock time-out on wait transaction(isc_lock_timeout).

    Диалект

    Обычно 1 или 3, SQL диалекты были введены в InterBase 6.0, для поддержки новых возможностей SQL, включая delimted идентификаторы. Допустимые диалекты:

      1 - Синтаксический анализатор обрабатывает по схеме InterBase V5.
      2 - Переходной флаг. InterBaseV6, and Firebird помечает неоднозначный SQL, создает и выдает ошибку или предупреждающее сообщение.
      3 - Синтаксический анализатор обрабатывает что - нибудь разграниченное одиночными кавычками как строковые константы и любое разграниченное двойными кавычками как SQL разграниченные идентификаторы.

    Цитируемый Идентификатор

    Эта опция гарантирует совместимость с базами данных, созданными в диалекте 1

    Чувствительный Идентификатор

    Эта опция заменяет атрибут SQL_IDENTIFIER_CASE (Значение по умолчанию SQL_IC_UPPER, выбирать SQL_IC_UPPER или SQL_IC_SENSITIVE)

    Автоцитируемый Идентификатор

    Значение по умолчанию NO (выбирать YES или NO)
    Детально: При выборе YES, следующий запрос должен измениться из

      SELECT A.Test_Field FROM Mixed_Caps_Table A
      ORDER BY A.Test_Field
    в
      SELECT A."Test_Field" FROM "Mixed_Caps_Table" A
      ORDER BY A."Test_Field"
    Обратите внимание: Если использовать следующий запрос, то конвертирование будет не верным!
    Входящий запрос
      Select A.Test_Field From Mixed_Caps_Table A
      Order By A.Test_Field
    результат
      "Select" A."Test_Field" "From" "Mixed_Caps_Table" A
      "Order" "By" A."Test_Field"

    "Выбор из списка" - работа с полем SCHEMA(OWNER)

      - "Поле SCHEMA не используется"
      - "Удалить поле SCHEMA из SQL запроса"
      - "Поле SCHEMA контролирует сервер"

    Некоторые приложения выполняют автоматическую генерацию SQL запросов. Например:
    select SYSDBA.COUNTRY,SYSDBA.CURRENCY from SYSDBA.COUNTRY
    или
    select * from SYSDBA.COUNTRY
    Для обеспечения совместимости с такими приложениями, служит этот список. Можно указать 3 варианта:

      "Поле SCHEMA не используется"
    - это значение по умолчанию. Это значит, что для фильтрации в процедуре SQLTables, Вы можете указать это поле, и на основании него будет правильно сформирована результирующая выборка, однако это поле будет установлено в NULL. Такое поведение было раньше, поэтому мы его сохраняем, для обеспечения совместимости.

      "Удалить поле SCHEMA из SQL запроса"
    - это значение, соответствует следующему правилу. Если на вход процедуры SQLExecDirect поступает запрос вида:
      select SYSDBA.COUNTRY,SYSDBA.CURRENCY from SYSDBA.COUNTRY
    то драйвер преобразует и передаст серверу на исполнение уже вот такой вариант:
      select COUNTRY,CURRENCY from COUNTRY

      "Поле SCHEMA контролирует сервер"
    - это значение предусмотрено на будущее. Тогда, когда сервер сам сможет обрабатывать такие запросы. В этом случае драйвер не контролирует, а сразу передает серверу на обработку.
