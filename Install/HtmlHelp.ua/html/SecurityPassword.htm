Захист Пароля

Під час запису DSN через ODBC Адміністратор пароль шифрується і записується в ODBC.INI. Але пароль допускається не вказувати при формуванні DSN. Він буде запрошений додатково під час з'єднання з базой даних або може бути переданий в стрічці з'єднання. Для отримання динамічного варіанту пароля, необхідно в операторі з'єднання SQLDriverConnect вказати буфер для вихідного параметра SQLCHAR *OutConnectionString, довжина якого повинна бути не менше 512, бажано 1024.
