Cursores

En el actual controlador Firebird ODBC los tipos de cursor Dynamic y Keyset serán modificados para utilizar el tipo Static, por lo tanto no es posible actualizar los sets. Para obtener mejor rendimiento utilice el tipo de cursor ForwardOnly. Operadores de lectura: (SQLFetch, SQLExtendedFetch, SQLScrollFetch) utilice SQL_ROWSET_SIZE y SQL_ATTR_ROW_ARRAY_SIZE. Para mayor performance al emplear campos tipo Blob use el operador SQLBindParameter (sin importar el tamaño del campo Blob) ya que ésto trabajará a mayor velocidad que utilizando SQLPutData/ SQLGetData.
